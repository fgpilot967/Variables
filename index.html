Erklärung von var, let and const

global scope und local scope

var ist function-scoped (local in der function), d.h. die var steht in der gesamten function und dem dazugeörigen block zür Verfügung

let und const sind block-scoped (local im einzelnen block), d.h. diese variablen stehen nur in dem jeweiligem block zur Verfügung

die variables die im global scope erstellt werden (im script allgemein) stehen allen zur Verfügung (inklusive innerhalb functions und blöcke)

wenn die variablen nicht im eigenen local bereich (block oder function) definiert sind, wird eine stufe up (parent) geschaut. also geht es aus dem block immer höher, bis letztendlich das global erreicht wird

im Beispiel 2 wurden die variablen in der function neu deklariert. Somit nehmen die variablen diesen wert an (auch im block, weil der block innerhalb der function als child ist)

im Beispiel 3 werden auch im block 2 neue variablen deklariert

im Beispiel 4 wird z im block neu definiert (deklariert) und steht NUR im block zur Verfügung. Ebenso wird x (var) neu deklariert (im block), und steht somit in der gesamten function zur Verfügung das ist die besonderheit einer var variablen
